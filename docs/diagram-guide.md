# Diagram Guide

Use the Excalidraw files as layered documentation.

## Reading Order

1. `ReadableSystemMap.excalidraw`
2. `CurrentSystemLayout.excalidraw`

The first file is the simple overview. The second file is the detailed map.

## What the Colors Mean

- blue: shell and grid zones
- green: companion and live-state flows
- pink: theme and appearance flows
- amber: recovery and validation flows

## What The Shapes Mean

- large framed boxes: major system areas
- smaller boxes: individual steps or rules
- arrows: data flow or sequence
- note boxes: direct guidance for implementation

## Rule Of Thumb

If a diagram becomes too dense to read in one glance, split the idea into a
new file instead of piling more content into the same canvas.
